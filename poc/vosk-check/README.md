# Vosk 精度検証 PoC（フェーズ1: 環境構築・素の精度測定）

## セットアップ手順（再現用）

```bash
cd poc/vosk-check
python3 -m venv venv
source venv/bin/activate
pip install vosk

# 日本語モデル（軽量版, 約48MB）をダウンロード
curl -L -o model-small-ja.zip https://alphacephei.com/vosk/models/vosk-model-small-ja-0.22.zip
unzip model-small-ja.zip
```

## 検証用データ

- `ground_truth.txt`: 表彰式を想定した読み上げ原稿（正解テキスト）
- `sample.wav`: 上記原稿を macOS の日本語音声合成（`say -v Kyoko`）で読み上げ、Vosk が要求する 16kHz/mono/16bit PCM に変換した音声（`.gitignore` 対象。再生成する場合は下記コマンド）

```bash
say -v Kyoko -o sample.aiff -f ground_truth.txt
afconvert -f WAVE -d LEI16@16000 -c 1 sample.aiff sample.wav
```

## 実行方法

```bash
source venv/bin/activate
python3 recognize.py sample.wav ground_truth.txt
```

## 所感・次のアクション（フェーズ1: 素の精度）

- CER 16.8% という数値のうち、実際に意味が変わってしまう誤認識（「功績」→「鉱石」、「表彰状の授与」→「洋書症状の需要」）は一部で、残りは漢字・送り仮名の表記ゆれによるもの。字幕として読む分には表記ゆれは大きな支障にならないが、**誤認識2件はどちらも今回の用途（表彰式）で重要な語**であり、無視できない
- この検証は **macOSのTTS合成音声**を使ったもので、はっきり発話されたクリーンな音声。実際の会場マイク・雑音環境ではCERはさらに悪化する可能性が高い点に注意

## 検証項目②: 辞書登録ありの精度検証（結果）

当初の想定は「Voskのgrammar機能に登録済みの人名・用語を投入すれば認識精度が上がる」だったが、検証の結果これは**部分的にしか成立しない**ことが判明した。

### Voskのgrammar機能

```python
grammar = json.dumps(["功績", "鉱石", "表彰状の授与", "洋書症状の需要", "[unk]"], ensure_ascii=False)
rec = KaldiRecognizer(model, wf.getframerate(), grammar)
```

- 「功績」vs「鉱石」のような**同音異義語の2択を絞り込む用途には有効**（正しく「功績」を選べた）
- 一方、**モデルの語彙に存在しない単語（`髙橋` のような外字を含む人名）を登録しても `Ignoring word missing in vocabulary` として無視される**。grammar機能は「モデルが元から知っている単語の中から絞り込む」仕組みであり、任意の新規単語を教え込むことはできない
- フレーズ全体（「表彰状の授与」）を1つの語として登録すると丸ごと無視される。単語単位でしか機能しない

→ **「事前学習資料から抽出した任意の人名をVoskに直接登録する」という当初の設計は、この方式では実現できない。**

### 読みベースの後処理補正 v1（`correct_by_reading.py`）— 文字レベルのスライド窓

Voskの生の認識結果（プレーンテキスト）に対し、登録済み用語の読み（`pykakasi`でひらがな変換）と照合し、近い部分文字列を置換する方式を試作。

| ケース | 結果 |
|---|---|
| 「鉱石」→「功績」（単一語の補正） | 成功 |
| 「表彰症状を用意」→「表彰状を授与」（複数語が近接） | 重複置換で不完全（「表彰状状」になった） |

単一語の補正は有効に機能したが、文字単位のスライド窓では語の境界を無視して誤ったオフセットにマッチすることがあり、複数語が近接するケースで破綻した。

### 読みベースの後処理補正 v2（`correct_by_reading_v2.py`）— Voskの単語区切り単位で照合

v1の反省を踏まえ、Voskが `SetWords(True)` で返す**単語分割**を最小単位にしてn-gram（1〜3語）で読みを照合する方式に改善。加えて、スコアが同点の場合は**大きい区間（複数語をまとめた置換）を優先**するルールを追加し、「表彰」「症状」をバラバラに「表彰状」で二重置換してしまう事故を解消した。

| ケース | v1 | v2 |
|---|---|---|
| 「表彰症状を用意」の「表彰症状」→「表彰状」 | 「表彰状状」（重複バグ） | 「表彰状」（正しく1回だけ置換） |
| 全体のCER（`sample.wav`, 「功績」のみ登録） | — | **16.8% → 15.1%**（辞書登録前後で比較） |

一方で、「用意」→「授与」の補正は v2 でも**発生しなかった**。理由を確認したところ、「用意（ようい）」と「授与（じゅよ）」は読みが大きく異なり、そもそも同音異義語ではない（Voskがまったく違う単語として聞き間違えた、より深刻な誤認識）。**読みベースの後処理補正が有効なのは「同音異義語・表記ゆれ」レベルの誤認識までであり、読みごと違う単語に化けるケースまでは救えない**、という射程の限界も分かった。

### 結論
**人名・専門用語の正確な表示は、Vosk側への辞書投入ではなく、単語分割ベースの読み照合による後処理補正で実現する。** v2の実装でアルゴリズムの重複置換バグは解消済み。ただし後処理補正は万能ではなく、「読みごと誤認識される」ケースは救えないため、事前学習資料からの用語抽出で**認識されやすい発音・言い回しに寄せる**運用上の工夫（例: 台本自体の言い回しを認識しやすい表現に事前調整する等）も合わせて検討する必要がある。`docs/app-design.md` を合わせて更新済み。
